Layout audit — before vs after

Two side-by-side viewport frames at 1280×560 (a common laptop size). Left frame: BEFORE — current production layout (rpanel 280px, the cards I added in Phase 1 and 2 don't fit). Right frame: AFTER — the proposed fix (rpanel 360px, map gives up ~80px). Each frame renders the same content with the same data so the size difference is what changes, not the content.

What this mockup proves:
1. Historical comparison (5-col table) — clips trade-impact text at 280px, fits cleanly at 360px.
2. UNCTAD scenarios — scenario column wraps awkwardly at 280px, single line at 360px.
3. India panel rows — sector flex column collapses to ~14px at 280px, readable at 360px.
4. Map at 1280px viewport — still 686px wide after the shrink, well above the 480px usability floor.
5. At 1024px laptop: map = 430px (was 510px). Still usable but tighter. If you want a softer trade-off, rpanel could be 340px (map = 450px) instead.

BEFORE · current production layout (rpanel 280px)

Three problems visible: (1) Historical comparison row "Hormuz closure (now)" — the trade-impact text "~20% global oil blocked" is cut off. (2) UNCTAD scenario column wraps onto two lines for longer scenario descriptions. (3) India ROW — sector column ("refiner" / "upstream" / "LNG") squeezed to ~14px between the ticker and the HIGH/MED tag.

BDTI cron fix — what the tile will read on each day

After the cron change (30 18 * * 5 → 0 14 * * 1-5), the scraper runs every business day after Baltic close (~1pm London). Below: what the BDTI tile will display each day given the cron + the label logic already in production.

BDTI tile — Today Mon 15:00 IST
2 310
▼ −2.7% · today's close
BDTI tile — Tue evening Tue 21:00 IST
2 285
▼ −1.1% · today's close
BDTI tile — Sat morning Sat 10:00 IST
2 285
▼ −1.1% · last publish Fri 22 May

Weekend tiles correctly read "last publish Fri" — that is real source cadence, not our staleness. On Mon-Fri evenings the tile reads "today's close". Currently (with the Friday-only cron), Mon-Thu tiles also read "last publish Fri" which is the bug.

1. rpanel 280 → 360 + map ~80px narrower
   Files touched: index.html (1 CSS rule change).
   Side effect: every existing card on the rpanel gets more horizontal room. Map narrower by ~80px on desktop.

2. BDTI cron weekly → daily-business-days
   Files touched: .github/workflows/bdti-weekly.yml (cron change). Optionally rename file to bdti-daily.yml.
   Side effect: scraper runs 5× per week instead of 1×. Stays under GHA scheduled-event limit by a wide margin.

3. Grid templates tightened on hc-row / un-row / ip-row + ellipsis on cells that can carry long strings
   Files touched: index.html (3 CSS rule tweaks).
   Side effect: cells truncate gracefully instead of overflowing. No visual change at typical text lengths.
