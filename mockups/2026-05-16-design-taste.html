Four design directions

Same 5 cards in every option (cross-chokepoint · verdict · market pulse · pipeline bypass · war-risk). Same data. Four different design languages. No sparklines. Bars, bullets, tables, stacked compositions, zone scales. Pick one, mix two, or veto a specific element — easier than describing taste from a blank page.

References: A = Stripe / Linear · B = Groww / Zerodha Kite · C = Bloomberg Terminal / n8n executions · D = Wall Street Journal / Publiq editorial
Option A
Editorial Minimal
Stripe / Linear. Maximum whitespace, mono numbers in light weight, thin hairline rules, one accent colour used sparingly. Bars are 2-3px tall and exist only to make a comparison visible.
Charts allowed: 2px horizontal bars, zone scales, ranked lists. Not used: stacked bars, gauges, icons, decoration.

Option C
Terminal Dense
Bloomberg / n8n exec view. Tables dominate. Tabular numbers, micro-bars inside cells, colour-coded backgrounds for risk states. Most metrics per square inch, intended for the user who keeps the dashboard open all day.
Charts allowed: in-cell micro-bars (~60px), heat-tinted cells, compact grids. Not used: hero numbers, large bars, big icons.

Option D
Visual Storytelling
Wall Street Journal / Publiq editorial. Each card is a small story: headline, sub-deck, the chart that proves it, an annotated callout. Serif headlines on dark; mono numbers inside the charts. Slower to scan but maximum interpretive payload.
Charts allowed: annotated bullet bars with baseline tracks, stacked composition, vertical thermometer, arc gauge. Not used: pure tables, micro-bars, generic pills.
