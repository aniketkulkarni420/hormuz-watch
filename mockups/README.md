# mockups/

Visual mockup files. One per UI change. Format: `YYYY-MM-DD-FEATURE.html`.

Each mockup must:
1. Show current (deployed) state
2. Show proposed state(s) — ≥2 options when there's a design decision
3. Be openable standalone in a browser (no build step)
4. Get user approval before the corresponding code changes land

See `.process/MOCKUP_FIRST.md` for the full rule.
