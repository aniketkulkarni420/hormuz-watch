## 4. HARD RULES (from prior corrections, do not violate)

1. **Cron cadence is data, not label** — if BDTI shows "Fri 15 May" Mon-Thu, it's because cron is `* * * * 5`, not because the label is wrong
2. **OilPriceAPI demo is live & free** — never call it "frozen demo data"; the freezing came from our KV throttling
3. **No new design language** — reuse `.rblock`, `.rhead`, existing row classes
4. **Compliance is non-negotiable** — Mode 2 cards need 3-layer + disclaimer + SEBI reg
5. **One commit per card / per concern** — never bundle layout + data + compliance in one push
6. **Test before push** — load `index.html` in browser, scan for clipping/overflow
7. **Show full HTML before push** — user wants to see the diff before it lands
8. **No recommendation grammar** — strip "trade-bias", "long X short Y", "you should"
9. **Verify with curl, not assumption** — when a source seems frozen, curl it directly
10. **Plain HTTP > Playwright** when site allows — StockQ, Stooq, OPA demo all work plain
11. **`totals.all = sum(by_type.values())`** in vessel scrape, not `max(unique_vessels)`
12. **Drift detect on every CSS change** — run `python scripts/check_mirror.py`
13. **One source of truth per KV key** — don't let two scrapers write same key (oil-stooq vs oil-web)
14. **Banner shows newest write, not freshest data** — removed the clobber, don't reintroduce
