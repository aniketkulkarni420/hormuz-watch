## 2. Architecture — the data flow

```
cron-job.org (external pinger)
  ├─ /api/admin/refresh   every 10 min → fans out 11 scraper workflows
  ├─ /api/selfheal        every 15 min → auto-remediation + incident tracking + escalation
  └─ /api/digest          daily 06:30  → readiness email (only when actionable)
        │
  GitHub Actions scrapers → write KV (OIL_KV)
        │  (each validates via _validate.py before write)
        ▼
  /api/snapshot, /api/oil, /api/bdti, /api/verdict, /api/integrity, ... read KV
        ▼
  index.html renders + confidence-gates tiles
```

GHA scheduled crons are **throttled ~10×** on free repos — that's why the cron-job.org pinger drives refresh, not GHA `schedule:`.

## 4. The self-correcting + self-improving system (built this session)

**Tier 1 — wrong data can't display:**
- `scripts/_validate.py` — shared sanity-bounds + anomaly-guard; scrapers reject garbage before write (wired into currency, aircraft, vessel; oil+bdti have bespoke guards)
- `/api/integrity` — PUBLIC validator: every feed scored fresh + sane + accurate → `showcase_ready` + per-feed `{status, confidence, reason}`. Catches "fresh but WRONG" (the WTI-$105, dark-200, $82-Brent bug class that freshness-only `/api/diag` missed)
- `index.html` integrity badge — "ALL SYSTEMS LIVE" / "N DEGRADED" + dims any suspect/dead tile

**Tier 2 — self-healing:**
- `/api/selfheal` — reads integrity, re-dispatches broken feeds' workflows via `GH_REFRESH_PAT`, tracks incidents in KV (`selfheal_state`), escalates CRITICAL feeds after 2 failed retries (deduped 6h), recovery notes paired to escalations, per-feed 20-min cooldown, `?dry=1` to test
- Builds `feed_health_7d` KV ledger (the system's memory) every run

**Tier 3 — daily digest (G):**
- `/api/digest` — CF endpoint (NOT GitHub — Resend key lives in CF). Pulls integrity + independent Brent-vs-Yahoo cross-check + 7-day reliability trends (names weakest feed). Emails **only when actionable**: not-ready / weak-feed / Monday weekly-summary / `?force=1`. Healthy days = silent (`suppressed_healthy`).

**Autonomy order before any email:** failover → last-good → retry → confidence-gate → THEN escalate.

## 5. Key design decisions / rules (do not break)

### Compliance (legal — SEBI)
- Disclaimer block + reg `INH000009843` must stay visible
- 3-layer India panel: **Data · Interpretation · Not Investment Advice**
- NO recommendation grammar ("buy/sell/hold/we recommend/target")
- No transactional affordances

### Brand / signature features (preserve)
- Cargo ticker ("$X transited today", live count)
- Verdict 4-level taxonomy: NORMAL <1.5 / ELEVATED 1.5-2.5 / HIGH 2.5-3.5 / CRITICAL ≥3.5
- Brent 4-week trend = Option A line+area+collision-safe markers (just shipped, 6-pass audit)
- Map (Leaflet) as geographic anchor
- "Last publish" dating on slow feeds (BDTI/EIA/OFAC)
- Honest degradation: `—` / STALE / WEB FEED / dim — never fake a number

### Technical constraints
- Single `index.html`, no SPA framework
- Cloudflare Pages + Pages Functions only
- Google Fonts only (Manrope + JetBrains Mono)
- Tabular numerals on all numbers (`font-variant-numeric: tabular-nums`)
- Dark theme only
- KV free tier = 1000 writes/day — we run ~980/day after 5-pass reduction (diff-write status, oil-scraper 2x/day, 30-min refresh window, etc.). DON'T add per-run KV writes carelessly.

### Accuracy-first principle (governs everything)
**Never show a number that could be wrong to a showcase visitor — degrade to "unavailable" instead.** This is why BDTI stayed single-source: all free 2nd sources (Hellenic editorial, TradingEconomics JS-only, investing.com frozen, Baltic paywalled) would inject garbage. A bad fallback is worse than none.

## 6. Hard rules / lessons (from this project's bug history)

1. **Cron cadence is data, not label** — stale display = fix the refresh, not the timestamp text
2. **`isFinite(null) === true`** in JS — use `Number.isFinite`. (Caused the WTI null.toFixed crash.)
3. **Freshness ≠ accuracy** — a fresh value can be wrong (stale-fallback, API cap, frozen constant). Validate both.
4. **Never render stale-as-live** — the WTI-$105 bug was EIA-weekly fallback shown as live. Kill silent stale fallbacks.
5. **API result caps mislead** — GFW limit:200 read as "200 dark vessels". Cap-aware labels ("200+").
6. **Cross-source agreement** before trusting (oil Stooq vs OPA).
7. **More ports without an anomaly guard HURTS** — a broken port adds 0, drops the sum. Guard rejects false drops.
8. **GITHUB_TOKEN can't dispatch sibling workflows** — use a fine-grained PAT (`GH_REFRESH_PAT`).
9. **Test in a real browser** — headless Playwright caught what code review + curl missed (the null crash, label overlaps).
10. **One commit per concern**, show diffs, get approval for design changes.
11. **Resend test sender (`onboarding@resend.dev`) only delivers to the account-owner email.** Verified domain needed for arbitrary recipients.
12. **Email should be exceptional, not routine** — system self-corrects silently; speaks only when action helps.
