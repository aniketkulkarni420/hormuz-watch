3. Full Right Sidebar — Design Inconsistency Audit

Issues I should have caught earlier. Each is independently fixable.
Vessel Type Mix · Inbound/Outbound tiles
In scrape mode, tiles say "5 Ports / 141 Vessels" but the label below says "Ports/Vessels" with same colors as AIS mode (blue/amber). Confusing carryover.
Either drop tiles entirely (Option C above) OR change colors to neutral grey when in port mode.
Vessel Movement · stale AIS placeholders
Direction split bar + east/west text + 3-tile grid all show 0s when AIS is down. Per-port bars rendered below as separate "afterthought".
When data_source = "web_scrape", HIDE the AIS placeholder elements via CSS class. Show only headline + per-port bars + banner.
Type bar labels · inconsistent widths
62px → 96px change made bars too narrow. Some labels still feel cramped at smaller sizes.
Set width:66px + use abbreviated labels (OFFSHORE, BULKER, CONTAINER, PASSENGER). Fits naturally.
BDTI freshness display
User saw "as of May 5" even after setting May 9. Suggests browser cache or stale frontend render.
Verify hard-refresh fixes it. If persistent, add cache-busting query param to /api/bdti fetch in dashboard JS.
Source badge color inconsistency
Some badges are green "LIVE FEED", some blue "WEB FEED", some amber "EIA WEEKLY". Mixed signals on what's actually fresh.
Standardize: green = live (less than 10 min), blue = recent (10 min - 24h), amber = historical (>24h), red = stale.
Tile padding inconsistency
Different tiles use slightly different padding (16/18 vs 14/16). Subtle but adds visual jitter.
Audit all .rblock instances. Standardize to 16px 18px.
Conclusion lines hidden by default
Several tiles have a .conclusion div with display:none — never gets populated. Dead code.
Either populate them OR remove the div. Currently confusing for next dev reading the code.
Tile section headers spacing
Section headers (.rsection) have inconsistent margin-top between sections (16px vs 24px).
Standardize section spacing to 20px or 24px.
