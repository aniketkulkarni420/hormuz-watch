Public API

Endpoint reference · base URL: https://hormuz-watch-7cd.pages.dev

Overview

The dashboard's data endpoints are publicly readable. They power the front-end UI but can also be consumed by downstream tools (research dashboards, alerting scripts, internal models).

All endpoints return JSON. CORS is open (Access-Control-Allow-Origin: *). No authentication is required for read endpoints.

Fair-use note: automated polling exceeding 1 request per minute per endpoint is discouraged and may be rate-limited. For higher-frequency or commercial use, please reach out — we provide service tiers with SLAs.

Endpoints

GET/api/oilpublic

Live Brent + WTI prices with tiered source fallback.

Response shape:

{
  "source": "Live commodity feed",
  "tier": "primary" | "secondary" | "tertiary-required",
  "brent": { "level": 108.42, "change": 0.94, "changePct": 0.87, "high": 109.1, "low": 107.8, "t": 1778... },
  "wti":   { "level":  95.18, "change": ..., "changePct": ..., ... },
  "fetchedAt": 1778...,
  "ageMin": 1.1
}

When tier: "secondary", the ETF proxy structure is returned instead — see source.

GET/api/stooqpublic

Daily reference Brent price (legacy endpoint name). Always-on fallback when live feed degrades. 1-2 trading day lag.

{ "today": 109.6, "yesterday": 108.4, "todayDate": "2026-05-08", "yesterdayDate": "2026-05-07" }
GET/api/eiapublic

EIA petroleum price series proxy. Used for weekly history (sparkline).

Parameters:

NameDescriptionExample
seriesEIA series codeRBRTE (Brent), RWTC (WTI)
lengthNumber of points to return6
frequencyweekly | daily | monthlyweekly
POST/api/gfwpublic

Satellite-detected vessel encounter + loitering events within the Hormuz geometry.

Body (JSON):

{
  "datasets": ["public-global-encounters-events:latest"],
  "startDate": "2026-04-05",
  "endDate":   "2026-05-05",
  "geometry":  { "type": "Polygon", "coordinates": [[[52,24],[59.5,24],[59.5,28.5],[52,28.5],[52,24]]] }
}

Returns { entries: [...] } with event objects.

GET/api/historypublic

Historical time series from Cloudflare D1. Hourly snapshots of every metric.

Parameters:

NameDescriptionExample
metricColumn name (whitelisted)brent_price, wti_price, bw_spread, transits_24h, dark_pct, bdti
rangeLookback window24h, 7d, 30d

Response:

{ "metric": "brent_price", "range": "7d", "points": [{ "ts": 1778..., "v": 108.42 }, ...], "count": 168 }
GET/api/snapshotpublic

Cached current-state snapshot. Used by downstream tools (e.g. India Risk Monitor) for the hormuz_throughput metric. Returns a stable schema across regime changes.

{
  "as_of": "2026-05-08T...",
  "daily_transit_estimate": 84,
  "vessel_count_inbound": 38, "vessel_count_outbound": 42,
  "pct_of_normal": 60.0,
  "dark_vessels": 947,
  "bdti": 14,
  ...
}
POST/api/recordtoken-auth

Hourly snapshot writer. Called by an internal cron job. Requires X-Snapshot-Token header. Not for public use.

External Worker endpoints

Specialised data sources hosted as separate Cloudflare Workers:

GEThormuz-ais-aggregator.aniket-kulkarni.workers.dev/statepublic

Authoritative server-side AIS aggregation. Single Durable Object subscribes to AISStream once, computes gate-crossings centrally so all users share the same numbers.

{
  "transits24h": 84,
  "eastbound24h": 41, "westbound24h": 43,
  "categories": { "transit": 12, "anchored": 8, "approach": 5 },
  "vesselCount": 25,
  "lastMsgAgeSec": 3,
  "wsConnected": true
}
GEThormuz-ais-aggregator.aniket-kulkarni.workers.dev/transitspublic

Recent gate-crossing events. Parameter ?range=24h (or 7d etc).

{ "count": 84, "transits": [{ "mmsi": ..., "name": "...", "dir": "eastbound", "time": ..., "lat": ..., "lng": ... }] }
